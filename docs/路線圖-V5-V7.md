## V5 —「評估 → 整合」資料橋接版（對應 OS P1 後段）

**一句話**：讓一個 Core 匯出檔，能在本站走完「評估 → 開局推演 → 復盤」全程，零後端。

| # | 目標 | 內容 |
|---|---|---|
| 5-1 | **沙盤劇本橋接** | evaluator 匯入的 owners[] 一鍵轉成 simulator **匿名開局盤面**：高持分未同意 → 🧱樞紐戶、opposed → 低起始滿意度、持分 → 影響力；經 localStorage 單向傳遞、不回寫 Core |
| 5-2 | **逐戶分回升級** | Core `calc_rights_exchange()` 落地後，改讀權威 `return_value`／`equalization`（找補金欄位＋紅黃燈），停用我方攤分示意 |
| 5-3 | **財務語言統一** | simulator 的示意財務改對齊 evaluator 同一套敘述（共負比、分回比、<70% 融資閘門），一詞一義（Phase 0「公式一致」的正式完成） |
| 5-4 | **難度誠實化** | 40 戶平衡微調（時間/籌碼隨戶數輕度放大）＋極端組合顯示 ★★★★★；全配置勝率回歸「難但可贏」帶（3–8/30） |

**完成標準**：拿一個合成案件 JSON，10 分鐘內完成「匯入評估 → 看逐戶分回 → 轉開局 → 推演 → 復盤報告」demo，全程無需解說文件。
**不做**：後端、AI、Dashboard 管理功能、修改 Core 公式。
**依賴 RE-DCF**：`calc_rights_exchange()`（5-2）；其餘無依賴。

## V6 — 管理層 Dashboard（對應 OS P2「Dashboard 只管理、不計算」）

**一句話**：開發部主管能用它管理 2–3 個併行案件的進度與同意率，不再開 Excel。

| # | 目標 | 內容 |
|---|---|---|
| 6-1 | **多案件庫** | 匯入多個 Core JSON 成案件清單（本機 localStorage），卡片顯示：階段（S1–S11 定位）、同意率 vs 門檻、共負比、警告數 |
| 6-2 | **同意進度追蹤** | 逐戶 consent 看板（匿名代號）＋時間軸；達標倒數（還差幾戶／幾 % 面積） |
| 6-3 | **時程甘特** | S1–S11 對映甘特圖（法定審議期標示），里程碑手動維護、無後端 |
| 6-4 | **檢核清單引擎** | 各階段產出物 checklist（取自 docs/開發流程架構.md）可勾選、進度隨案件存 |
| 6-5 | **KPI 總覽** | 跨案件儀表：總銷合計、平均共負比、風險警告彙總——**全部數字只讀 Core `result`，Dashboard 一條公式都不寫** |

**完成標準**：3 個合成案件併行管理 demo；任何顯示數值都能追溯到來源 JSON 的欄位；主管首次使用不需教學即可看懂案件狀態。
**不做**：財務／容積／共負公式（SSOT 紅線）、真實姓名地址（CRM 只用匿名代號＋本機備註）、GIS 正式圖資（最多靜態示意）。
**依賴 RE-DCF**：無新依賴（消費既有 1.1 契約）；期望對方 `warnings[]` 規則持續擴充。

## V7 — 智慧層 Urban-Renewal-Agent（對應 OS P3／RE-DCF Phase 2）

**一句話**：AI 建立在穩定的 Domain Model 之上，而不是取代它——**獨立新 repo**，不動本站靜態純度。

| # | 目標 | 內容 |
|---|---|---|
| 7-1 | **Urban-Renewal-Agent 立庫** | 獨立 repo；FastAPI 包 RE-DCF Core → 提供計算端點，開通**雙向回算**（Dashboard 權利人資料 → 回算權變） |
| 7-2 | **AI 地主** | LLM 人格化談判對手接進 simulator「訓練模式」：自然語言談判、依 PERS 人格回應、可復盤對話策略 |
| 7-3 | **AI Copilot** | 案件摘要／法規助手（RAG on 法規資料庫）／財務解讀（讀 Core result 講人話） |
| 7-4 | **多 Agent 協作** | 法規歸法規、計算歸計算、合約歸合約（README 既定分工）自動化編排：一個案件 JSON 進，八段標準報告出 |

**完成標準**：AI 回答可溯源（引法條號／Core 欄位）、計算永遠出自 Core 端點（AI 不心算）、simulator 訓練模式一場完整 AI 談判可玩可復盤。
**不做**：AI 直接給投資建議（維持 [FRAME] 邊界）、繞過 Core 的任何估算。
**開工 Gate（三個都要）**：① V5+V6 穩定運轉；② Core Domain Model 補齊（權變／找補／IRR／NPV／現金流／法規資料庫——對方 ROADMAP P1）；③ 真實清冊資料的隔離方案定案（LLM 不落地真實 PII）。

## 優先序與依賴總覽

```
V4.x 收尾 ──► V5 橋接（依賴：calc_rights_exchange）
                └──► V6 管理層（無新依賴，純消費 1.1 契約）
                        └──► V7 Agent（Gate：V5+V6 穩定 × Core Domain Model × 資料隔離）
```

- **本站（Urban-Renewal）到 V6 為止永遠是純前端靜態站**；所有後端與 AI 集中到 V7 的新 repo。
- 每一版收尾都要過同一套關卡：headless 迴歸測試、資安自查（無外傳／無密鑰／noopener）、資料紀律稽核（零真實案名金額）、README 同步。
