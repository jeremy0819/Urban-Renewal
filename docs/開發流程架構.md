# 都更／危老 開發流程架構（全生命週期 S1–S11）

> 重審版。把「拿到一塊地」到「交屋成立管委會」切成 11 個階段，每階段定義**目的・產出物・檢核點・負責 agent**。
> 用途：任何案件不論大小、不論走到哪個進度，先在本座標**定位階段**，再用該階段檢核點逐項核對——即「確認並梳理」的標準動作。
> 通用方法論，不含真實案件資料。版本 1.0｜2026-06。

## 0. 階段定位法（先做這個）

> **TL;DR**：看案夾結構 → 找最新關鍵檔 → 對照下表取「最高已達階段」＝目前進度。

1. **看案夾結構**：前期評估夾常見編號 `00地主提供／01謄本／02投報分析／03建築師／05顧問評估／06清冊·意願／07說明會／08合約／10整合評估`；進入正式執行後轉為生命週期夾 `00行政／05營運管控／08事業計劃／10整合評估／12審查／15規劃設計／20採購發包／25施工／30銷售`。
2. **找最新關鍵檔**：蹤催檢核表、會議記錄、說明會／審議文件的日期與版本。
3. **對照下表**取「最高已達階段」＝目前進度；該階段的檢核點就是當下要核的事。

| 案夾訊號 | 推定階段 |
|---|---|
| 只有謄本／權狀 | S1 |
| 有投報分析、建築師面積表 | S2–S4 |
| 有說明會夾（邀請函・簽到・簡報） | S5 |
| 有合約夾且有簽回件 | S6 |
| 有事業計畫／估價作業／審查夾 | S7–S8 |
| 有發包・施工圖・營造合約 | S9–S10 |
| 有銷售・交屋文件 | S11 |

> ⚠️ 啟發式判讀僅供初篩，仍須開最新會議記錄確認（例如「有合約夾」可能只是擬稿未用印）。

## 1. 前期評估（S1–S4）

> **TL;DR**：地能不能做（S1）→ 能蓋多少（S2）→ 能賣多少坪（S3）→ 划不划算（S4）。四步走完才知道值不值得整合。

### S1 基地與權屬調查 — 地能不能做
- **產出**：基地基本資料表、權屬清冊（含**所有權人人數**）、都更／危老／防災路徑初判。
- **檢核**：✓ 是否位於已劃定更新地區（否則需自行劃定單元）✓ 屋齡・使照有無 ✓ 違建／海砂／輻射 ✓ 臨路寬深。
- **Agent**：`urban-renewal-analysis`（案夾盤點）＋ `urban-renewal-law-assistant`（路徑初判）。

### S2 法規與容積試算 — 地能蓋多少
- **產出**：容積帳明細（法定＋獎勵拆項＝允建容積）、同意比例門檻與現況差距。
- **檢核**：✓ 獎勵率拆項與法源對齊（都更上限、危老上限、地方加碼均查現行條文）✓ §162 三項免計**逐層**核算（勿用 FA×% 總量法）✓ FA 用**使照面積**非謄本 ✓ 同意門檻依劃定方式判（自行劃定門檻較高）。
- **Agent**：`urban-renewal-law-assistant`（條文查證）→ **RE-DCF-Tool**（容積查核，黃金測試鎖定）。

### S3 坪效與量體規劃 — 能賣多少坪
- **產出**：銷售坪、公設比、戶數車位配置、銷坪比。
- **檢核**：✓ 銷坪比落住宅正常區間 1.58–1.68 ✓ 公設比用「專有/(1−n)」回推驗證 ✓ 車位數跨過 50 輛即觸發雙車道（影響地下效率）。
- **Agent**：RE-DCF-Tool。

### S4 投報財務評估 — 划不划算
- **產出**：投報總表（總銷／共同負擔六大科目／分回／報酬率）＋敏感度。
- **檢核**：依[投報分析架構-正確版](投報分析架構-正確版.md)四區塊逐塊核；✓ 投報只讀坪效輸出不重算容積 ✓ 共負比＞65%（分回＜35%）需檢視地主接受度 ✓ 三來源（建築師／代書／建設）銷售坪 ±2% 收斂。
- **Agent**：RE-DCF-Tool ＋ 四角色快覽（建管／建築師／地政士／開發商）。

## 2. 整合（S5–S6）

> **TL;DR**：把投報數字搬上桌（S5 說明會、同源同版）→ 白紙黑字定下來（S6 簽約，合約類型與投報模式一致）。

### S5 說明會與意願整合
- **產出**：說明會紀錄與簽到、意願統計（同意比例 vs 門檻差距）、逐戶分配試算（簡報檔清冊）。
- **檢核**：✓ 對外簡報數字與內部投報同源同版 ✓ 逐戶分配回算等於總分回 ✓ 意願統計分母用「所有權人數＋面積」雙軌。
- **Agent**：`urban-renewal-analysis`（進度盤點）＋ RE-DCF-Tool（分配試算）。

### S6 簽約
- **產出**：用印合約（合建／雙軌／委託實施／整合委任／全案管理擇一）、簽約率統計。
- **檢核**：✓ 合約類型與投報模式一致（投報用全案管理、合約卻是合建＝重大不一致）✓ 管理費率・撥付階梯・自動解約門檻逐條核 ✓ 簽約率達門檻才往 S7。
- **Agent**：`contract-reviewer`（風險條款）。

---

## 3. 法定程序（S7–S8）

> **TL;DR**：送件給政府審（S7，前期投報參數升級為送審值）→ 過審拿核定函（S8，修正後一改全案連動回寫）。

### S7 事業計畫／權利變換送審
- **產出**：事業計畫書、權變計畫（估價師三家）、共同負擔表（送審版）、拆補與安置計畫。
- **檢核**：✓ 共負科目對齊法定六類 ✓ 前期投報參數升級為送審值（營業稅足額、估價師單價取代行情假設）✓ 拆補・租金補償有估價依據。
- **Agent**：`urban-renewal-law-assistant`＋ RE-DCF-Tool（共負連動核對）。

### S8 審議・公展・核定
- **產出**：審查意見回覆、公展紀錄、核定函。
- **檢核**：✓ 審議修正後回寫投報（容積／共負一改全案連動）✓ 異議戶處理紀錄完備。
- **Agent**：`urban-renewal-analysis`（追蹤審議里程碑）。

---

## 4. 執行（S9–S11）

> **TL;DR**：畫圖找營造（S9）→ 動工蓋（S10，變更設計影響容積者回 S2 重核）→ 賣屋交屋收尾（S11，實際售價回測敏感度）。

### S9 規劃設計與發包
- **產出**：建照、施工圖、發包比價表、營造合約。
- **檢核**：✓ 建照核准容積 vs 核定計畫一致 ✓ 發包價 vs 投報營建單價差異回寫敏感度。
- **Agent**：`contract-reviewer`（營造合約）。

### S10 施工
- **產出**：開工備查、營運管控（進度・請款・變更設計）、融資動撥紀錄。
- **檢核**：✓ 變更設計影響容積者回 S2 重核 ✓ 請款進度 vs 現金流量表。

### S11 銷售・交屋・管委會
- **產出**：銷售去化報表、使照、交屋清冊、管委會成立。
- **檢核**：✓ 實際售價回測投報敏感度 ✓ 找補結算 vs 權變核定值。

## 5. 跨階段鐵律

> **TL;DR**：兩層不打架、單一數源、改一動全、查證優先、唯讀分析——五條貫穿全流程。

1. **兩層不打架**：容積查核（坪效層）是唯一量體真相；投報層只讀其輸出。改容積回 S2，改價格成本在 S4。
2. **單一數源**：對外簡報、合約附件、送審文件的數字必須同源於同一版投報；版本以日期戳記。
3. **改一動全**：任一上游修正（容積、單價、費率）必須沿 S2→S4→S5 分配→S7 共負 全鏈更新。
4. **查證優先**：法規數字一律標來源條號並對照現行條文；行情一律標查證日期。
5. **唯讀分析**：對案夾的盤點分析全程唯讀，報告寫到案夾之外。

## 名詞速查

| 名詞 | 速解 |
|---|---|
| **坪效層／容積查核** | 從基地 FA 算到銷售坪的量體計算；本體系唯一「量體真相」(oracle)。詳見[投報分析架構](投報分析架構-正確版.md#1-兩層邊界最高鐵律)。 |
| **投報層** | 由銷售坪算總銷→共同負擔→分回／報酬率；只讀坪效輸出，不重算容積。 |
| **§162 三項免計** | 機電／梯廳／陽台依規定免計入容積的部分；超出須**逐層**補計入（非 FA×% 總量法）。 |
| **共同負擔** | 都更權變中由全案分攤的六大科目（工程／管維／權變／利息／稅捐／管理）。 |
| **銷坪比** | 銷售坪 ÷ 允建坪，住宅正常區間 1.58–1.68，偏離即須查公設比與免計。 |
| **同意比例門檻** | 啟動更新所需的所有權人／面積同意比例；自行劃定單元門檻較高。 |
| **使照面積 vs 謄本面積** | FA 一律用**使照**基地面積；用謄本會墊高免計上限、掩蓋超容。 |
| **4D** | DECONSTRUCT 拆解 / DIAGNOSE 診斷 / DEVELOP 構建 / DELIVER 交付，見 [README](../README.md#ai-協作方法論4d)。 |
