# Spec：互動式開發儀表板 + 既有 docs 優化（2026-06-17，已核可）

## 目標
在不重組既有架構的前提下，優化 README 與兩份 docs 的可讀性，並新增一個**單一自含 HTML 互動式開發儀表板**（`index.html`），讓任何人 30 秒看懂都更／危老開發評估體系，並能互動定位案件階段。介面以**有趣且易懂**為主。

## 交付物
- D1 `index.html`：單一自含 HTML（inline CSS＋原生 JS，零依賴、零建置），可離線開、可上 GitHub Pages。含 6 個互動模組：
  1. 🛤️ 流程跑道 S1–S11（點站台展開 目的/產出物/檢核點/負責 agent，四色階段帶）
  2. ⚙️ 兩層引擎（坪效 oracle → 投報，「只讀不重算」單向邊界演示）
  3. 🦸 Agents 戰隊（翻牌卡片）
  4. 🔄 4D 循環環（點象限看都更落點）
  5. 🩺 投報踩坑健檢（10 項可勾選＋健檢分數儀表，遊戲化）
  6. 🎯 案件階段定位器（勾選訊號→算出目前 S 階段＋下一步）
- D2 `README.md` 優化：30 秒看懂開場、置頂導覽列、四色 mermaid、顯眼連到儀表板。
- D3 兩份 docs 優化：每節一行 TL;DR、互鏈、名詞速查；不動原結構。

## 風格
藍圖藍（#2563eb）＋安全琥珀（#f59e0b）＋核可綠（#16a34a）＋法定紫（#7c3aed）。圓角卡片、emoji、流暢過場與輕動畫、手機可開。文案活潑但精準。

## 邊界
- 全通用、**無真實案件資料**（案名／金額／基地數字），遵守 `.gitignore` 鐵律。
- 內容資料一律取自既有兩份 docs，不另造數字。
- 既有 docs 就地優化，不重組結構。

## 驗收
- `index.html` 直接雙擊可開、手機可開、6 模組皆可互動。
- README 首頁渲染四色架構圖並顯眼連到儀表板。
- 洩漏掃描通過（無案名／大額數字）。
- push 至 `claude/urban-renewal-dashboard-docs-ab2euy`；Pages 由該分支或合併後啟用。
